# Demo 2 說明：英翻中

## 它在幹嘛？

讓 AI 把英文翻譯成中文。
這個任務叫做 **Machine Translation（機器翻譯）**。
Google 翻譯、DeepL 都是這類技術，只是規模大很多。

## 跟 Demo 1 比，多了什麼？

Demo 1：
```python
classifier = pipeline("sentiment-analysis")   # ← HuggingFace 自己選模型
```

Demo 2：
```python
translator = pipeline(
    "translation",
    model="Helsinki-NLP/opus-mt-en-zh",       # ← 我們自己指定模型
)
```

**為什麼要自己指定？**
因為翻譯有「方向」：
- `opus-mt-en-zh` 是英 → 中
- `opus-mt-zh-en` 是中 → 英
- `opus-mt-en-ja` 是英 → 日
- ……

HuggingFace 沒辦法猜你想翻哪個方向，所以要你講清楚。

## 模型名字的格式

`Helsinki-NLP/opus-mt-en-zh` 拆開來看：

| 部分 | 意思 |
|---|---|
| `Helsinki-NLP` | 發佈者（赫爾辛基大學 NLP 團隊） |
| `/` | 分隔符 |
| `opus-mt-en-zh` | 模型名（OPUS 資料集訓練、英 → 中） |

格式很像 GitHub 的 `用戶名/專案名`。
你可以去 https://huggingface.co/Helsinki-NLP 看他們發的所有翻譯模型。

## 預期輸出

```
=== AI 英翻中結果 ===

原文：Hello, world!
譯文：哈囉,世界!

原文：I am learning artificial intelligence.
譯文：我正在學習人工智能.

原文：Taiwan is a beautiful island.
譯文：臺灣是一個美麗的島嶼.

原文：Programming is easier than people think.
譯文：規劃比人們想象的容易.
```

⚠️ 注意最後一句 — `Programming` 被翻成「規劃」而不是「程式設計」。
**這是真實的 AI 缺點**：開源小模型在一字多義時常常猜錯。
商用大模型（ChatGPT、DeepL）通常會翻對。

這也是為什麼學 AI 要學「**怎麼選模型**」，不只是「**怎麼跑模型**」。

## 玩玩看

1. **換句子**：在 `english_sentences` 加你自己的英文
2. **換方向**：把 `opus-mt-en-zh` 改成 `opus-mt-zh-en`，再餵中文句子進去
3. **換語言**：HuggingFace 搜尋 `opus-mt-en-XX`（XX 是兩字母語言碼），試試英翻日、英翻法、英翻德

## 名詞小抄

| 名詞 | 中文 | 一句話解釋 |
|---|---|---|
| `translation` | 翻譯（任務名） | 把一種語言變成另一種 |
| `model` | 模型 | 訓練好的 AI（檔案 + 參數） |
| `Helsinki-NLP` | 模型發佈者 | 像 GitHub 的「使用者帳號」 |
| `en-zh` | 英文到中文 | 翻譯方向 |
| `translation_text` | 翻譯後的文字 | AI 回傳的欄位名稱 |
