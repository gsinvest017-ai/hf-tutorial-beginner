# Demo 3 說明：圖片辨識

## 它在幹嘛？

讓 AI 看一張圖片，告訴你裡面是什麼。
這個任務叫 **Image Classification（圖像分類）**。
這是電腦視覺（Computer Vision）裡最基本的任務之一。

## 跟前兩個 Demo 比，最神奇的一件事

**你會發現用法幾乎一樣！**

| Demo | 程式碼 |
|---|---|
| Demo 1（文字） | `pipeline("sentiment-analysis")` |
| Demo 2（文字） | `pipeline("translation", model="...")` |
| Demo 3（圖片） | `pipeline("image-classification", model="...")` |

文字、圖片、聲音、影片……都可以用 **同一個 `pipeline()`**。
這就是 HuggingFace 設計的厲害之處：**抽象化**。
你不用為每種任務學新的 API。

## 預期輸出

```
=== 正在辨識圖片 ===
圖片網址：http://images.cocodataset.org/val2017/000000039769.jpg

=== AI 的猜測（信心度從高到低）===

  第 1 名：Egyptian cat
          ███████████████████████████ 90.2%

  第 2 名：tabby, tabby cat
          █ 4.1%

  第 3 名：tiger cat
          █ 2.4%

  第 4 名：lynx, catamount
          ▏ 0.6%

  第 5 名：Siamese cat, Siamese
          ▏ 0.5%
```

第 1 名是「Egyptian cat（埃及貓）」，信心 90%。
其他 4 名也全部是貓的品種 ── AI 雖然不一定 100% 答對品種，但「**這是貓**」這件事 99% 確定。

這個現象叫做「**分布合理（reasonable distribution）**」── 即使猜錯第 1 名，
前幾名也都在合理範圍內。你可以拿來判斷模型有沒有發瘋。

## 一個你應該知道的小細節

這個模型只能辨識**1000 種**事先定義好的物體（叫做 ImageNet-1k 類別）。
如果你給它一張：
- 「你家狗狗」 → 它會告訴你品種，可能對也可能錯
- 「你的早餐拉麵」 → 它可能說「soup bowl」或「pasta」
- 「某個 cosplay 人物」 → 它可能完全認不出，只說「mask」或「costume」

**這個叫做 closed-set classification（封閉類別分類）**。
更厲害的模型（CLIP、BLIP）可以辨識「**任何你用文字描述的東西**」 ── 那叫 open-vocabulary。
之後想學可以查 `CLIP zero-shot classification`。

## 玩玩看

### 玩法 1：換圖
把 `image_url` 換成任何網路圖片的網址。
要找測試圖可以：
- Google Images 隨便抓一張
- 用 https://picsum.photos/600/400 隨機圖

### 玩法 2：用你電腦裡的圖
把網址換成本地路徑：
```python
image_url = r"C:\Users\你的名字\Pictures\my_dog.jpg"
```
（Windows 路徑前面要加 `r` 不然反斜線會出事）

### 玩法 3：批次辨識
丟一個清單進去，一次處理多張：
```python
results = classifier([
    "http://image-1.jpg",
    "http://image-2.jpg",
    "C:/local/image-3.png",
], top_k=3)
```

## 名詞小抄

| 名詞 | 中文 | 一句話解釋 |
|---|---|---|
| `image-classification` | 圖像分類 | 看圖分類成 N 個類別之一 |
| `ViT` | Vision Transformer | 把處理文字的 Transformer 拿來看圖 |
| `ImageNet-1k` | 一個資料集名 | 1000 類、120 萬張圖的標準訓練集 |
| `top_k` | 前 K 名 | 取最有可能的前幾個答案 |
| `closed-set` | 封閉類別 | 只能從事先定好的清單裡選 |
| `open-vocabulary` | 開放詞彙 | 可以辨識任何文字描述的東西 |
